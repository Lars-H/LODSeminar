Request

Desciption:

The compare method can be used in order to compare a given value and unit to another resource. The available resource can be found here. Our algorithm decides on a division factor in order to find a comparable object.

Accepts:

Acceptable Datatypes: "application/ld+json"
For all other Accept-Types: HTML is returned.

Parameters:

  • v(required)

    • Type: float

    Example: 12993

  • u(required)

    • Type: enum

    Unit: [t,k,g,km,m,euro,usd]

    Example: "t"

Example

The following example request a value of "2000000" and a unit of "t" (tons):

 http://cepler-1157.appspot.com/compare?v=2000000&u=t

  • Example
  • Response
  • GET

Request

Desciption:

The compare method can be used in order to compare a given value (v), unit (u) and a range (r) with a resource on DBpedia. The range specifies a relative tolerance around the actual value. Please note that, there will be no division factor used. (Depending on the range it is (almost) 1:1)

Accepts:

Acceptable Datatypes: "application/ld+json"
For all other Accept-Types: Not acceptable is returned

Parameters:

  • v(required)

    • Type: float

    Example: v=12993

  • u(required)

    • Type: enum

    Unit: [t,k,g,km,m,euro,usd]

    Example: "u=t"

  • r(required)

    • Type: float

    Limits: r in [0,1]

    Example: "r=0.2"

Example

The following example request a value of "2000000", a unit of "t" (tons) and a range of 20%. Here, cURL is used in order to specify the Accept Header.

curl -v -H "Accept: application/ld+json" "http://cepler-1157.appspot.com/dbpedia/compare?v=2000000&u=t&r=0.2"

/worldbank/compare
  • Request
  • Example
  • Response
  • GET

Request

Desciption:

The compare method can be used in order to compare a given value (v), unit (u) and a range (r) with a resource on the Worldbank. The range specifies a relative tolerance around the actual value.
Please note that only monetary units (US-Dollar and Euro) are supported. The data is stored in US-Dollar, therefore, we try to get the latest exchange rate using the Yahoo Finance API. Please note that, there will be no division factor used. (Depending on the range it is (almost) 1:1)

Accepts:

Acceptable Datatypes: "application/ld+json"
For all other Accept-Types: HTTP Status code 406 (Not Acceptable) is returned.

Parameters:

  • v(required)

    • Type: float

    Example: v=12993

  • u(required)

    • Type: enum

    Unit: [euro,usd]

    Example: "u=usd"

  • r(required)

    • Type: float

    Limits: r in [0,1]

    Example: "r=0.2"

Example

The following example request a value of "300", a unit of "usd" (US-Dollar) and a range of 40%. Here, cURL is used in order to specify the Accept Header.

  curl -v -H "Accept: application/ld+json" "http://cepler-1157.appspot.com/worldbank/compare?v=300&u=usd&r=0.4"
